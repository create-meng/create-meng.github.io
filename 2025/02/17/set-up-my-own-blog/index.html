运行本地博客

在open git bash here下运行并且点击（复制）链接到浏览器

如图则成功打开hexo默认界面,路径为http://localhost:4000/

bug2:
本地打开未设置端口

该bug为Ctrl+路径时没有打开端口，此时复制地址到浏览器打开即可

3.hexo相关命令

  • hexo clean # 清除缓存文件，建议写完文章后执行一次
  • hexo g # 生成 public 文件夹，写完文章执行
  • hexo s # 启动 hexo 服务
  • hexo new “文章标题” #创建新的文章，在_post下
  • 配置完文件/写完博客执行 hexo g

配置博客界面

打开_config.yml，配置以下

- suthor ：作者

  • 其他的根据喜好配置即可，不赘述

配置博客主题

hexo官方主题页面
链接为hexo的官方主题页面 部分主体已不可使用

  1. 点击Visit preview site预览该主题，点击主题名称进入该主题下的github

  2. 选择好看实用的主题下载压缩包（Next、hexo-theme-Chic-master…）

  3. 解压后将文件包放在themes下，并且修改_config.yml的theme名称

  4. 修改主题的_config.yml

  • 名称
  • 图标
  • 作者
  • 联系方式等等

创建新仓库

仓库名必须为 （用户名）.github.io，否则可能无法将本地文件上传至GitHub，这个命名方式是 GitHub Pages 的特殊要求，用于识别和部署个人网站或博客。
仓库为公开仓库

配置git

配置ssh

配置git的ssh（由于linux有了远程，所以需要重新配置ssh防止覆盖）
ssh-keygen -t rsa -b 4096 -C “1375528118@qq.com“ -f “D:\Git\liminghao.ssh\custom_key”
-t 加密方式
-b 生成密钥位数
-C 邮箱
-f 保存位置
中途可以不需要密码，直接按回车

安装hexo-deployer-git插件

在刚刚创建的博客文件夹下再次打开 git bush，安装 hexo-deployer-git 插件

配置deploy

打开主界面的_config.yml，在最底下配置deploy
deploy:
type: git
repo: git@ssh.github.com:CREATE-MENG/create-meng.github.io.git
branch: main
YAML 文件对缩进非常敏感，deploy 下的子项（如 type、repo、branch）必须保持一致的缩进，通常使用两个空格。

2.从 cacert.org 下载最新的 CA 证书文件，更新git的SSL证书

3.repo为HTML链接，改为SSH链接即可，即上述内容
